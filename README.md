## Introduction
**MoodMate** is a high-fidelity interactive UI/UX prototype developed as part of the **ISB 36403 - Interaction Design** course at **Universiti Kuala Lumpur (UniKL)**. The project demonstrates the implementation of user-centered design principles through a practical mental health and wellness management application.

The goal of this project is to provide a hands-on application of interaction design operations — empathizing with users, prototyping, and usability testing — while delivering a user-friendly graphical experience built with Figma.

## Problem Statements & Objectives

### Problem Statements
* **Restrictive & Inaccessible Design**: Existing mental health apps limit utility for users who cannot afford premium subscriptions and exclude visually impaired users with poor contrast.
* **Confusing Interfaces & Lack of Guidance**: Users often fail to interpret their progress effectively due to confusing graphs and feel overwhelmed by insufficient onboarding.
* **Lack of Action Control**: Users experience frustration when unable to reverse or modify inputs, such as undoing actions submitted to an AI.

### Objectives
* Implement a fully functional **user-centered design** to manage a mental wellness app.
* Apply UI/UX principles including **Shneiderman's Eight Golden Rules**, **Feedback**, **Visibility**, and **Affordance**.
* Build an interactive **Figma-based high-fidelity prototype** with seamless navigation.
* Evaluate usability using the **System Usability Scale (SUS)** to gather actionable user feedback.

## Program Scope
The MoodMate Prototype allows users to:
* **Add and Track Moods** through daily check-ins using emoji-based selections.
* **Review Health Data** with categorized reports and vitals monitoring (heart rate, weight, blood group).
* **Explore Content** with personalized recommendations, trending videos, and goal-based browsing filters.
* **Communicate** with mental health professionals in real-time via the secure SafeSpace chat interface.

## Information Architecture
The prototype is implemented using a categorized card-based structure and a persistent bottom navigation bar to manage different user flows.

`[Authentication] -> [Home Dashboard] <-> [My Health] <-> [Discover] <-> [SafeSpace] <-> [Profile]`

### Module Structure
* **Authentication**: Entry point for manual and social logins.
* **Home Page**: Core hub for personalized recommendations and check-ins.
* **My Health**: Detailed reports and vitals tracking.
* **SafeSpace**: Real-time professional support messaging.

| Module | Responsibility |
| :--- | :--- |
| `Authentication` | Handles manual sign-up, social logins, and password recovery |
| `Home / Discover` | Displays personalized content, quick access cards, and interactive browsing |
| `My Health` | Manages categorized health reports, vitals tracking, and educational articles |
| `SafeSpace` | Provides a secure interface for real-time text-based chat with experts |
